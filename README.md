# 🛡️ TruthTrace — 网络事件全维度溯源与辟谣平台

**追溯每一条信息的源头。分析传播链路。识别谣言。还原真相。**

---

## 产品定位

TruthTrace 是一款面向公众的信息可信度检测工具。输入任何网络链接（网页/微博/知乎/B站/抖音），系统自动追溯信息源头，通过10维度推理引擎进行全维度分析，生成学术风格的溯源报告，帮助用户识别虚假信息和误导性内容。

### 核心能力

- **10引擎推理管线** — 失真检测·逻辑谬误·统计滥用·拼接造谣·5层溯源·领域知识·叙事框架·模态漂移·综合判定·纠偏建议
- **多平台溯源泉** — 微博/知乎/微信公众号/B站/抖音/快手/新闻网站
- **学术风格报告** — 完整摘要·方法·结果·讨论·参考文献·推理链附录
- **权威知识验证** — 19条可引用知识，6领域权威来源，每条附带URL和证据等级
- **实时监控** — 多平台热点自动爬取+分析+叙事告警
- **浏览器扩展** — 一键分析·页面浮窗·转发前警告

### 关键原则

- **可溯源** — 每个结论附带证据片段和来源URL
- **不确定就说** — 超出知识范围的内容返回"unverifiable"，不编造答案
- **不创造知识** — 知识验证仅基于已知权威来源
- **透明推理链** — 完整推理过程对外开放

## 快速启动

### 零依赖模式（SQLite，立即可用）

```bash
# 后端
cd backend
pip install -r requirements.txt
DATABASE_URL="sqlite+aiosqlite:///../truthtrace_local.db" uvicorn app.main:app --port 8000

# 前端
cd frontend
npm install
npm run dev
```

访问: http://localhost:5173
API文档: http://localhost:8000/api/docs

### Docker 模式（完整功能：PostgreSQL + Redis + Celery）

```bash
docker-compose up -d
```

需要 Docker Desktop (Windows需开启虚拟化)。

## 架构

```
浏览器扩展 ─┐
Web前端 ────┤
            ├── FastAPI (86路由) ──┬── 10引擎推理管线
            │                      ├── 安全防线(5层)
            │                      ├── 质量监控
            │                      ├── 自我进化引擎
            │                      └── 爬虫集群
            │
            ├── PostgreSQL (数据库)
            ├── Redis (缓存/队列)
            ├── Celery (定时任务)
            └── Elasticsearch (可选)
```

## 引擎管线

| # | 引擎 | 检测内容 |
|---|------|---------|
| 1 | 失真检测 | 7种模式 (源头伪造/内容篡改/错误引用/忽略语境/情感操纵/权威绑架/语境剥离) |
| 2 | 逻辑谬误 | 12种 (因果倒置→选择性证据) |
| 3 | 统计滥用 | 8种 (相对/绝对风险/样本量/混杂/基线率) |
| 4 | 拼接造谣 | 5种 (逻辑跳跃/因果缝合/时间线操控) |
| 5 | 5层溯源 | L1表面→L2内容→L3来源→L4叙事→L5利益 |
| 6 | 领域知识 | 6领域 (食品/医药/经济/法律/环境/历史) |
| 7 | 叙事框架 | 12种 (阴谋论→虚假平衡) |
| 8 | 模态漂移 | 5种 (推测→确定/意见→事实/条件→绝对) |
| 9 | 综合判定 | 可信度评分 + 多维融合 |
| 10 | 纠偏建议 | 教育性解释 + 科学素养提示 |

## 安全

- L1 输入净化 (防注入/XSS/命令注入)
- L2 爬虫沙箱 (SSRF防护/10MB限制/危险内容类型屏蔽)
- L3 恶意内容检测 (脚本/iframe/phishing)
- L4 安全头 (CSP/HSTS/X-Frame-Deny)
- L5 速率限制 + CSRF + 蜜罐
